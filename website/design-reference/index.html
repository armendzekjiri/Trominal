Why Trominal

Built for the way infra teams actually work.

Closed-source SSH clients hold your hosts hostage. Trominal flips that — your data sits on your server, your team shares one source of truth, and every line of code is on GitHub.

Self-hosted by default

Run the Trominal server in a Docker container next to your other infra. Your hosts, keys, and snippets never leave your network.

Team-first sync

Share host configs and snippets across the team without copy-pasting ~/.ssh/config. Role-based access, audit logs, the works.

AI that respects context

Inline ghost-text suggestions trained on your shell history — running fully on-device or against your own LLM endpoint. No prompts shipped to vendors.

Mobile that doesn't suck

A custom keyboard purpose-built for SSH. Pipe, tilde, ctrl, escape — all one tap away. Snippets and AI are right above the keys.

End-to-end encrypted vault

Identities and secrets are sealed with a master password before they hit the wire. Even your server admin can't read them.

Open source, MIT-licensed

Audit it, fork it, patch it. We ship binaries, but the truth is on GitHub. No EULA, no rug-pulls, no enterprise tier.

The terminal, reimagined

Hosts, snippets, files, tunnels — one window, one keyboard shortcut.

Tabs and splits you'd expect, plus a sidebar that actually understands your infrastructure: grouped by environment, color-coded by tags, searchable by anything.

Get started in 2 minutes

One container. One CLI. Three platforms.

Run the server with Docker, point your client at it, and you're connected. The same vault works on macOS, Linux, Windows, and iOS.

Installation guide Read the docs
~/infra
# Run the Trominal server
$ docker run -d \
    -p 8443:8443 \
    -v trominal-data:/var/trominal \
    ghcr.io/trominal/server:1.4.2
→ trominal v1.4.2 listening on :8443
 
# Pair the desktop client
$ brew install trominal/tap/trominal
$ trominal pair https://trominal.acme.internal
